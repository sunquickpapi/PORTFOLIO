Collaborative E-Commerce

A multi-role web platform designed to streamline the production of bespoke jerseys. Unlike standard retail systems, it integrates a specialized workflow connecting Customers, Agents, and Designers. Features include a real-time negotiation chat, a design draft approval system, and role-specific dashboards to manage the lifecycle of a custom order from concept to final production.

My Role: Lead Full-Stack Developer

The Challenge

The primary difficulty lay in facilitating custom apparel orders that required complex coordination between four distinct parties: Customers, Agents, Designers, and Admins. Relying on fragmented tools for communication and manual tracking created a disconnected workflow where critical design feedback was often lost between stakeholders. This lack of a centralized system made it nearly impossible to track sales analytics efficiently or manage individual designer queues without errors, highlighting a critical need for a unified digital solution to handle the iterative nature of bespoke jersey production.

The Solution

To address this, LVNASTUDIO was engineered using VS Code as the development environment, utilizing XAMPP for local server management and SQL to handle the intricate database relationships required for a multi-role ecosystem. The platform features specialized dashboards for each user type, including a "Custom Your Own" module for customers and a collaborative chat interface where Agents can mediate design changes between Customers and Designers. This architecture ensures that every design iteration and approval is securely recorded in the database, seamlessly connecting the front-end customization tools with the back-end order management system.

🏆

Results & Impact

The implementation of this centralized platform successfully streamlined the entire production workflow, significantly reducing miscommunication and errors in custom orders. By integrating real-time analytics for Admins and task management tools for Designers, the system improved operational visibility and efficiency. The shift from manual processes to this digital ecosystem ensured that sales data, design drafts, and customer feedback were synchronized, leading to higher order accuracy and a faster turnaround time for custom jersey deliverables.

💡

Key Learnings

This project provided deep insights into the importance of structuring SQL databases to support effective Role-Based Access Control (RBAC), ensuring that Admins, Agents, and Designers only access data relevant to their specific workflows. Developing with VS Code and XAMPP highlighted the technical value of integrating communication tools directly into the transaction flow rather than relying on external apps. It demonstrated that a well-architected database is essential for maintaining data integrity across a complex, multi-user environment where real-time collaboration is the core feature.
